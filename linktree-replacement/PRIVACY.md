# Privacy Commitment

This LinkTree replacement package is designed with privacy as a core principle.

## What This Package Does NOT Do

- **No data collection** - We don't collect any information about you or your visitors
- **No tracking pixels** - No hidden tracking or fingerprinting
- **No cookies** - No cookies are set (unless you add analytics)
- **No third-party services** - No external APIs or services required
- **No phone-home** - The code never contacts any external server
- **No analytics** - No usage data sent to us, ever

## What This Package DOES Do

- **Runs entirely in the browser** - All code executes client-side
- **Stays in your Framer account** - Your content lives on Framer's servers
- **Respects user preferences** - Components honor `prefers-reduced-motion`
- **Works offline** - Once loaded, no internet required for basic functionality

## Analytics Are 100% Opt-In

The analytics documentation (GA4/GTM setup) is provided for users who WANT to track their own data. This is:

- **Completely optional** - You never have to add any analytics
- **Your data** - If you add GA4, the data goes to YOUR Google account
- **Your choice** - We don't require or encourage any tracking

## What Framer Collects

Note that while this package collects nothing, Framer (the hosting platform) has its own privacy policy:

- Framer may collect standard web server logs
- Framer may use cookies for their platform features
- See [Framer's Privacy Policy](https://www.framer.com/privacy/) for details

This is outside our control - it's part of using Framer as a hosting platform.

## Comparison to LinkTree

| Feature | LinkTree | This Package |
|---------|----------|--------------|
| Collects visitor data | Yes | No |
| Tracks clicks | Yes | Only if you add GA4 |
| Requires account | Yes (theirs) | Yes (your Framer account) |
| Data ownership | LinkTree owns | You own |
| Third-party sharing | Yes | No |
| Open source | No | Yes (MIT) |

## Our Commitment

We will never:

1. Add tracking to this package
2. Require external service connections
3. Collect data about users of this package
4. Monetize in ways that compromise privacy

## Questions?

If you have privacy concerns, please [open an issue](../../issues) and we'll address them transparently.
